# AgeCare Family — Mobile PWA

React 18 + Vite 5 PWA for family caregivers. Installs to home screen on iOS/Android.

## Running

```bash
# From repo root (pnpm workspaces)
pnpm --filter @agecare/mobile dev      # http://localhost:5174

# Or inside mobile/
pnpm dev
```

Backend must be running on port 3000. The Vite dev server proxies all `/auth`, `/elders`, `/alerts`, `/push`, and `/ws` requests.

## Build

```bash
pnpm --filter @agecare/mobile build
pnpm --filter @agecare/mobile preview
```

## Testing push notifications locally

1. Open Chrome, navigate to `http://localhost:5174`
2. Enable the PWA service worker: open DevTools → Application → Service Workers → verify the SW is active.
3. Log in with any seed user (e.g. `david@agecare.demo` / `agecare-demo-2026`).
4. Accept the notification permission prompt.
5. To simulate a push without a real VAPID server:
   - Open DevTools → Application → Service Workers
   - In the "Push" text field enter a JSON payload:
     ```json
     {"title":"AgeCare Alert","body":"Fall detected","alertId":"test-123","severity":"critical"}
     ```
   - Click **Push**
6. A native notification should appear; clicking it navigates to `/alerts/test-123`.

### iOS Safari (≥16.4 on iOS 16.4+)

1. Open `http://localhost:5174` in Safari (must be HTTPS in production; for local dev use a self-signed cert or ngrok).
2. Tap **Share → Add to Home Screen**.
3. Open the installed PWA from the home screen (push only works when launched as a standalone app on iOS).
4. iOS 16.4+ supports Web Push; earlier versions will see the in-app fallback banner.

## Architecture notes

- **State**: Zustand auth store, persisted to `localStorage` via `zustand/middleware/persist`.
- **Data fetching**: TanStack Query v5 with 30s refetch for vitals, 15s for alerts.
- **Realtime**: WebSocket at `/ws/client?token=` — reconnects automatically on disconnect. `alert.new` frames show an in-app toast even when the OS push doesn't fire (works on desktop dev).
- **Push**: VAPID public key fetched from `/push/vapid-public-key`. Subscription POSTed to `/push/subscribe`. Idempotent.
- **Service worker**: Generated by `vite-plugin-pwa` (Workbox). Runtime caching: network-first for `/api/*`, stale-while-revalidate for static assets and Google Fonts.
- **Icons**: Placeholder green-square PNGs at `public/icon-192.png` and `public/icon-512.png`. Replace with proper artwork before production.
